# Claude Chat Extension v0.8.2 Release Notes

**Версия**: 0.8.2  
**Дата релиза**: 09 января 2025  
**Тип релиза**: Улучшение UX (User Experience)

---

## 🎯 Основные изменения

### 1. **Упрощённая система состояний (2 вместо 4)**

Теперь расширение использует только 2 понятных состояния:

- **🟢 READY** - "Готов к запросу" 
  - Ассистент ждёт вашего сообщения
  - Можно отправлять новые запросы

- **🔄 WORKING** - "Ассистент работает"
  - Обрабатывает ваш запрос
  - Пишет ответ
  - Выполняет действия (tool calls)
  - Анимированная крутилка показывает активность

### 2. **Убраны эмодзи из табов**

- В табах теперь отображается только имя сессии
- Эмодзи состояний остаются только в заголовке окна чата
- Более чистый и минималистичный интерфейс

### 3. **Анимированный индикатор работы**

- Вместо статичных эмодзи теперь используется анимированная крутилка
- CSS анимация вращения для состояния WORKING
- Визуально понятно, когда ассистент активно работает

## 🔧 Технические изменения

### Обновлённые файлы:

1. **Система состояний**:
   - `ClaudeState.ts` - enum с 2 состояниями вместо 4
   - `ClaudeStateDetectionFacade.ts` - новые эмодзи (🟢 и 🔄)
   - `ClaudeStateManager.ts` - упрощённая логика переходов
   - `JsonlPatternDetector.ts` - адаптированная детекция

2. **UI компоненты**:
   - `TabBar.tsx` - убран импорт и отображение SessionStateIndicator
   - `SessionStateIndicator.tsx` - поддержка 2 состояний
   - `SessionStateIndicator.css` - анимация вращения для крутилки
   - `ChatWindow.tsx` - обновлённые типы состояний

## 🐛 Известные проблемы

### **Отслеживание tool calls** (будет исправлено в v0.8.3)
- Система пока не отслеживает продолжение работы ассистента после текстового ответа
- После ответа ассистента состояние переходит в READY через 3 секунды, даже если ассистент продолжает выполнять tool calls
- Это будет исправлено в следующей версии

## 📊 Сравнение с предыдущей версией

### Было в v0.8.1:
