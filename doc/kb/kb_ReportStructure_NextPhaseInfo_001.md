# База знаний: Структура отчетов - Информация о следующей фазе

**Дата создания:** 07.07.2025  
**Версия:** 001  
**Категория:** Documentation, Reports  

---

## Обязательное требование

**В конце каждого отчета сессии ДОЛЖНА быть секция с информацией о следующей фазе.**

---

## Структура секции "ПЛАН НА СЛЕДУЮЩУЮ СЕССИЮ"

### Обязательные элементы:

```markdown
## 📋 ПЛАН НА СЛЕДУЮЩУЮ СЕССИЮ

### 🎯 **ЗАДАЧА X.X: Название фазы**
**Команда:** `/команда --параметры --persona-название`

### **Ожидаемые результаты:**
- Конкретный результат 1
- Конкретный результат 2
- Конкретный результат 3
```

---

## Пример правильного оформления

```markdown
## 📋 ПЛАН НА СЛЕДУЮЩУЮ СЕССИЮ

### 🎯 **ЗАДАЧА 7.4: Терминальная синхронизация**
**Ответственный специалист:** Backend Developer (`--persona-backend`)

### **Команда для запуска следующей сессии:**
```bash
/build --feature "terminal synchronization" --persona-backend
```

### **Ожидаемые результаты:**
- Улучшенная bidirectional коммуникация между webview и extension
- Type-safe интерфейсы для сообщений
- Comprehensive error handling система
- State synchronization между компонентами
```

---

## Почему это важно

1. **Преемственность**: Следующий специалист точно знает что делать
2. **Автоматизация**: Команда готова к копированию и запуску
3. **Ожидания**: Четкие критерии успеха следующей фазы
4. **Планирование**: Возможность оценить объем работ
5. **Отслеживание**: Прогресс проекта виден на каждом этапе

---

## Где размещать

**Позиция:** В самом конце отчета, после всех других секций, но перед финальным заключением.

**Порядок секций в отчете:**
1. Выполненные задачи
2. Список измененных файлов  
3. Текущее состояние проекта
4. Возникшие проблемы и их решения
5. Метрики сессии
6. **📋 ПЛАН НА СЛЕДУЮЩУЮ СЕССИЮ** ← здесь!
7. Финальное заключение

---

## Ответственность

**Каждый специалист обязан:**
- Четко определить следующую фазу
- Указать нужного специалиста
- Подготовить точную команду запуска
- Сформулировать ожидаемые результаты

---

**Применение:** Все отчеты сессий
