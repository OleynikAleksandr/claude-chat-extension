# Release v0.12.6 - Enhanced Service Status Bar

## <¨ UI Improvements

### Dynamic Status Display
The service status bar now provides real-time, context-aware status updates based on Claude's JSON messages:

#### Status Types
1. **"Assistant Ready For Next Task"** - Displayed only when `type: "result"` is received (static text)
2. **"Reading user request"** - When `type: "system"` is detected
3. **"Sending requested information"** - When `type: "tool_result"` is detected
4. **Tool-specific statuses** - For all tool usage with format: `[ToolName]: [file/path/url]`

#### Supported Tools
All Claude tools are now properly displayed with their target files/paths:
- Task
- Bash
- Glob
- Grep
- LS
- Read
- Edit
- MultiEdit
- Write
- NotebookRead
- NotebookEdit
- WebFetch
- TodoWrite
- WebSearch

### Visual Enhancements
- **Pulsating animation** - Status text pulses while assistant is processing
- **Static display** - "Assistant Ready For Next Task" remains static
- **Marquee scrolling** - Long file paths scroll horizontally for better visibility
- **Persistent status** - Previous status remains visible until next update arrives

### Behavior Rules
- Status updates only when new type/name is detected in JSON stream
- Processing states show pulsating animation
- Ready state shows static text
- Long paths are truncated with scrolling marquee effect
